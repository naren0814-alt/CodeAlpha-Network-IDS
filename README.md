# Network Intrusion Detection System (NIDS)

## Project Overview

This project implements a basic Network Intrusion Detection System (NIDS) using Snort and Python. The system continuously monitors network traffic, detects suspicious activities using custom Snort rules, generates alerts, stores intrusion logs in a database, automatically blocks suspicious IP addresses, and visualizes detected attacks using graphs.

This project satisfies the following requirements:

- Set up a network-based intrusion detection system using Snort.
- Configure rules and alerts to detect suspicious activity.
- Monitor network traffic continuously.
- Implement response mechanisms for detected intrusions.
- Visualize detected attacks using graphs.

# Features

## Network Traffic Monitoring

The system continuously monitors incoming and outgoing network traffic using Snort.

## Intrusion Detection

The following attacks are detected:

- ICMP Ping Detection
- SSH Connection Attempts
- FTP Access Attempts
- Port Scanning Activity

## Alert Generation

Whenever suspicious activity is detected, Snort generates an alert.

## Database Logging

Detected attacks are stored in an SQLite database with:

- Timestamp
- Attack Type
- Source IP Address

## Automated Response

The system automatically blocks suspicious IP addresses using iptables.

## Attack Visualization

Detected attacks are visualized using a bar graph generated by Matplotlib.

# Snort Configuration

Copy custom rules:

```bash
sudo cp rules/custom.rules /etc/snort/rules/
```

Open Snort configuration:

```bash
sudo nano /etc/snort/snort.conf
```

Add the following line:

```text
include $RULE_PATH/custom.rules
```

Save and exit.

---

## Verify Snort Configuration

```bash
sudo snort -T -c /etc/snort/snort.conf
```

Expected output:

```text
Snort successfully validated the configuration
```

---

# Network Interface Configuration

Identify your network interface:

```bash
ip a
```

Example:

```text
eth0
```

or

```text
ens33
```

or

```text
wlan0
```

Use your interface name in the next command.

---

# Start Snort Monitoring

Example:

```bash
sudo snort -A fast -q -c /etc/snort/snort.conf -i eth0
```

Replace `eth0` with your interface.

Snort will now monitor network traffic continuously.

# Attack Simulation


```text

```

```bash

```

Alert generated:

```text
SSH Attempt
```

---

```bash

```

Alert generated:

```text
FTP Attempt
```

---

## Port Scan Detection

```bash
nmap -sS <TARGET-IP>
```

Alert generated:

```text
Port Scan Detected
```

---

# Automatic Response Mechanism

When a suspicious IP address is detected:

1. Alert is generated.
2. Event is stored in database.
3. Source IP address is extracted.
4. Attacker IP is blocked automatically using iptables.

Example:

```text
[ALERT] Port Scan Detected

[BLOCKED] 192.168.1.100
```

---

# Database Logging

The system stores intrusion information in:

```text
database/intrusion.db
```

Stored fields:

| Field | Description |
|---------|-------------|
| id | Attack ID |
| timestamp | Detection Time |
| attack_type | Type of Attack |
| source_ip | Attacker IP Address |
